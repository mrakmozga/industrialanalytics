# IndustrAI — интерактивный мониторинг промышленного оборудования

IndustrAI — это учебный/демо-проект на Flask + Socket.IO для визуализации телеметрии оборудования, оценки риска инцидента и имитации работы оператора в реальном времени. Приложение читает исторические данные из CSV-файлов и воспроизводит их как поток событий на веб-дашборде. 

## Что умеет приложение

- Отображает в реальном времени 5 сенсоров:
  - температура,
  - вибрация,
  - давление,
  - нагрузка,
  - влажность.
- Показывает вероятность инцидента (`risk`) и уровень опасности (норма / наблюдение / внимание / критично).
- Считает производные признаки:
  - `temp_pressure_ratio`,
  - `vibration_load_interaction`.
- Поддерживает управление симуляцией:
  - старт, пауза/возобновление,
  - сброс,
  - скорость 1× / 2× / 5×.
- Ведёт ленту предупреждений/инцидентов и обновляет графики по потоку событий Socket.IO.

## Технологии

- **Backend:** Flask, Flask-SocketIO, Eventlet, Pandas.
- **Frontend:** HTML/CSS/JS, Chart.js, Socket.IO client.
- **Контейнеризация:** Docker, Docker Compose.

## Быстрый старт

### Вариант 1: Локально (Python)

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

После запуска откройте: `http://localhost:5000`.

### Вариант 2: Docker Compose

```bash
docker compose up --build
```

После старта приложение будет доступно на `http://localhost:5000`.

## Переменные окружения

Поддерживаются следующие переменные:

- `DATA_CSV` — путь к CSV с телеметрией (по умолчанию `data/equipment_simulation_with_anomalies_25000.csv`).
- `PREDICTIONS_CSV` — путь к CSV с предсказаниями риска (по умолчанию `data/predictions.csv`).
- `HOST` — адрес bind для Flask-SocketIO (по умолчанию `0.0.0.0`).
- `PORT` — порт приложения (по умолчанию `5000`).

## Формат входных данных

### Файл телеметрии (`DATA_CSV`)

Ожидаются как минимум следующие колонки:

- `temperature`
- `vibration`
- `pressure`
- `load_factor`
- `humidity`

### Файл предсказаний (`PREDICTIONS_CSV`)

- Если есть колонка `risk`, используется она.
- Иначе используется первая колонка файла.
- Значения должны быть числовыми и интерпретируются как вероятность инцидента в диапазоне `[0..1]`.

Если длины файлов различаются, сервер берёт минимальную длину из двух.

## Socket.IO протокол

### События от клиента к серверу

- `start` — начать симуляцию.
- `pause` — поставить на паузу.
- `resume` — продолжить.
- `reset` — сбросить индекс симуляции в 0.
- `set_speed` — установить скорость (`{ "multiplier": 1|2|5 }`).
- `start_after_reset` — старт после сброса.

### События от сервера к клиенту

- `ready` — начальная готовность (`{ total }`).
- `tick` — новая точка потока с сенсорами, риском и производными признаками.
- `sim_done` — симуляция закончилась (`{ total }`).
- `reset_ok` — подтверждение сброса (`{ total }`).

## Логика симуляции

- Для каждого WebSocket-клиента хранится отдельное состояние сессии (`index`, `speed`, `playing`, `running`).
- Фоновая задача отправляет `tick` с задержкой, зависящей от выбранной скорости:
  - `1×` → `1.0s`
  - `2×` → `0.5s`
  - `5×` → `0.2s`
- При достижении конца датасета отправляется `sim_done` и воспроизведение останавливается.

## Типичные проблемы и диагностика

- **Ошибка загрузки CSV:** проверьте пути `DATA_CSV`/`PREDICTIONS_CSV` и наличие файлов в `data/`.
- **Нет подключения к Socket.IO:** убедитесь, что сервер запущен и порт `5000` доступен.
- **Пустой график/нет тиков:** проверьте, что `risk`-файл содержит числовые значения и что длина датасета больше 0.
