## 1. Обзор системы

IndustrAI считывает поток телеметрии из датасета (имитация датчиков), передаёт показания в ML-сервис для классификации риска в реальном времени и отображает результаты на интерактивном дашборде.

### Архитектура

Система состоит из трёх независимых компонентов, запускаемых через Docker Compose:

```
┌─────────────────────────────────────────────────────────┐
│  Браузер  →  Dashboard (Flask/SocketIO :5000)           │
│                        ↓  POST /predict                  │
│              ML Service (FastAPI :8000)                  │
│                        ↓  joblib.load                    │
│              Random Forest .pkl + StandardScaler .pkl    │
└─────────────────────────────────────────────────────────┘
```

| Компонент | Технологии | Порт |
|---|---|---|
| ML Service | FastAPI, scikit-learn, joblib | 8000 |
| Dashboard Backend | Flask, Flask-SocketIO, eventlet | 5000 |
| Frontend | HTML/JS, Chart.js, Socket.IO | — |

### Поток данных

| Шаг | Компонент | Действие |
|---|---|---|
| 1 | Backend | Читает строку из CSV (температура, вибрация, давление, нагрузка, влажность) |
| 2 | Backend → ML | `POST /predict` с сырыми показаниями сенсоров |
| 3 | ML Service | Масштабирует (StandardScaler), вычисляет производные признаки, возвращает `risk` |
| 4 | Backend → Frontend | WebSocket `tick` с показаниями + risk |
| 5 | Frontend | Обновляет графики, датчики, счётчики, принимает решения по режиму |

## 2. ML-сервис

### 2.1 Модель

| Параметр | Значение |
|---|---|
| Алгоритм | Random Forest Classifier |
| Деревьев | 500 |
| Классы | 0 — норма, 1 — инцидент |
| Версия sklearn | 1.3.0 (совместимость с 1.5+ обеспечена патчем `monotonic_cst`) |
| Формат | joblib (.pkl) |

### 2.2 Признаки (7 штук)

Модель принимает 5 масштабированных показаний сенсоров и 2 производных признака:

| Признак | Описание | Формула |
|---|---|---|
| `temperature_scaled` | Масштабированная температура | `(T − 70.54) / 6.09` |
| `vibration_scaled` | Масштабированная вибрация | `(V − 3.06) / 0.53` |
| `pressure_scaled` | Масштабированное давление | `(P − 99.90) / 8.05` |
| `load_factor_scaled` | Масштабированная нагрузка | `(L − 0.700) / 0.099` |
| `humidity_scaled` | Масштабированная влажность | `(H − 40.05) / 10.10` |
| `temp_pressure_ratio` | Отношение T к P (скал.) | `T_s / P_s` |
| `vibration_load_interaction` | Взаимодействие V и L | `V_s / L_s` |

### 2.3 Вычисление риска

`predict_proba()` возвращает вектор `[p₀, p₁]` — голосование деревьев. Риск вычисляется как нормализованная вероятность инцидента:

```
risk = p₁ / (p₀ + p₁)
```

Значение `risk ∈ [0, 1]` интерпретируется тремя уровнями:

| Диапазон | Статус | Действие |
|---|---|---|
| 0 – 29% | НОРМА | Продолжать работу |
| 30 – 49% | ПРЕДУПРЕЖДЕНИЕ | Запланировать ТО в 24–48 ч |
| ≥ 50% | ИНЦИДЕНТ | Немедленная остановка оборудования |

### 2.4 API

```
GET  /health           — состояние сервиса и модели
POST /predict          — инференс для одной записи
POST /predict/batch    — пакетный инференс
GET  /docs             — Swagger UI
```

**Запрос:**
```json
POST /predict
{
  "temperature": 75.1,
  "vibration": 3.2,
  "pressure": 98.4,
  "load_factor": 0.71,
  "humidity": 45.0
}
```

**Ответ:**
```json
{
  "risk": 0.37,
  "confidence": 0.82,
  "features_used": ["temperature_scaled", "..."],
  "feature_values": { "temperature_scaled": 0.748, "..." }
}
```

### 2.5 Fallback-режим

При недоступности ML-сервиса backend автоматически переключается в режим `cached` — использует предварительно рассчитанный `predictions.csv`. В хедере дашборда отображается индикатор: **ML LIVE** (голубой) или **ML CACHED** (жёлтый).

## 3. Дашборд

### 3.1 Структура интерфейса

| Зона | Содержимое |
|---|---|
| Левый сайдбар (260px) | Датчики оборудования (5 гейджей), управление графиками (вкл/выкл + масштаб), производные признаки |
| Центральная область | Счётчики событий, временной ряд риска (линии 30% и 50%), мини-графики сенсоров |
| Правая панель (300px) | Круговой индикатор риска, рекомендации ИИ, журнал событий |

### 3.2 Пороговые значения

Рассчитаны по реальному датасету — 5-й и 95-й процентили нормальных записей (24 250 строк без инцидентов):

| Параметр | Норма (5–95%) | Предупреждение | Инцидент |
|---|---|---|---|
| Температура | 62 – 78°C | За пределами нормы | > 85°C |
| Вибрация | 2.3 – 4.7 mm/s | За пределами нормы | > 6.1 mm/s |
| Давление | 85 – 105 кПа | За пределами нормы | < 70 или > 110 кПа |
| Нагрузка | 0.43 – 0.97 | За пределами нормы | > 1.0 |
| Влажность | 28 – 62% | За пределами нормы | > 70% |

### 3.3 Режимы работы

| Режим | Поведение |
|---|---|
| 👁 Наблюдение (по умолчанию) | Система отображает данные и алерты, не вмешивается в работу |
| 🤖 Активный | При риске ≥ 50% система автономно ставит симуляцию на паузу. Оператор перезапускает вручную кнопкой «Перезапустить оборудование» |

### 3.4 Управление

| Элемент | Действие |
|---|---|
| `1×` / `2×` / `5×` | Скорость воспроизведения (1, 2, 5 тиков/с) |
| ⏸ Пауза / ▶ Продолжить | Приостановить и возобновить поток |
| ↺ Сброс | Вернуть к началу датасета, обнулить счётчики |
| Переключатели графиков | Скрыть/показать каждый из 5 графиков |
| `−` / `+` масштаб | Зум оси Y с шагом ×1.5 |

### 3.5 Журнал событий и экспорт CSV

Кнопка **⬇ CSV** скачивает лог в формате CSV (разделитель `;`, UTF-8 с BOM для Excel).

Каждый алерт об инциденте содержит кнопку 🔍 — при нажатии открывается popup с полными показаниями сенсоров в момент инцидента.

**Структура CSV:**

| Колонка | Описание |
|---|---|
| `timestamp` | Дата и время записи |
| `index` | Номер точки в датасете |
| `temperature / vibration / pressure / load_factor / humidity` | Показания сенсоров |
| `risk_pct` | Вероятность отказа в % |
| `status` | НОРМА / ПРЕДУПРЕЖДЕНИЕ / ИНЦИДЕНТ |
| `type` | ТЕЛЕМЕТРИЯ или СОБЫТИЕ |
| `note` | Описание для системных событий |

> **Важно:** лог хранится в памяти браузера и очищается при обновлении страницы. Максимум 25 000 строк.

## 5. Запуск

### 5.1 Требования

- Docker Desktop (или Docker Engine + Compose plugin)
- Папка `models/` с тремя `.pkl` файлами
- Папка `data/` с двумя `.csv` файлами

### 5.2 Команды

```bash
# Первый запуск
docker compose build --no-cache
docker compose up

# Последующие запуски
docker compose up

# Полная пересборка после изменения файлов
docker compose down --rmi all
docker compose build --no-cache
docker compose up

# Пересборка только одного сервиса
docker compose up --build dashboard
```

### 5.3 Адреса

| URL | Назначение |
|---|---|
| `http://localhost:5000` | Дашборд (основной интерфейс) |
| `http://localhost:8000` | ML-сервис API |
| `http://localhost:8000/docs` | Swagger UI документация ML API |
| `http://localhost:8000/health` | Статус ML-сервиса |

### 5.4 Переменные окружения

| Переменная | Сервис | По умолчанию | Описание |
|---|---|---|---|
| `ML_MODE` | Dashboard | `live` | `live` — инференс через API, `cached` — из CSV |
| `ML_URL` | Dashboard | `http://ml:8000` | Адрес ML-сервиса |
| `DATA_CSV` | Dashboard | `data/equipment_simulation_...csv` | Путь к телеметрии |
| `PREDICTIONS_CSV` | Dashboard | `data/predictions.csv` | Fallback предсказания |
| `MODEL_PATH` | ML | `models/incident_random_forest_model.pkl` | Путь к модели |
| `SCALER_PATH` | ML | `models/scaler.pkl` | Путь к скейлеру |
| `FEATURES_PATH` | ML | `models/model_features.pkl` | Имена признаков |
